# Milestone v1.0: 门诊诊鉴智能体 MVP

**Status:** ✅ SHIPPED 2026-03-26
**Phases:** 1-5
**Total Plans:** 14

## Overview

v1.0 交付了一个可运行的门诊诊鉴 MVP：既包含面向医生的嵌入式实时诊鉴入口，也包含面向管理者的统计分析首页、问题分布洞察、病例下钻与导出链路。整个 milestone 围绕“统一指标口径 + 实时评估沉淀 + 管理闭环”展开，先锁定 PostgreSQL 语义层和时间窗口规则，再逐步叠加首页分析、医生端诊鉴、持久化复用以及 release hardening。

## Phases

**Goal**: 建立支撑实时诊鉴和首页统计的统一数据模型、时间口径和指标计算逻辑  
**Depends on**: Nothing (first phase)  
**Plans**: 3 plans

Plans:

**Details:**

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**

- Decision: 将 PostgreSQL 语义函数和事实模型作为指标单一来源，避免前端或 Python 层重复定义口径
- Decision: 医生端诊鉴通过 EMR host seam 嵌入，不接管宿主路由
- Decision: 实时诊鉴使用同步 POST + 10 秒预算 + 统一状态容器，保证医生端交互稳定
- Decision: 管理端 drilldown 和 CSV export 复用同一问题病例查询辅助函数，避免屏幕与导出集合漂移
- Decision: 生产前角色边界先以宿主注入 `viewer role` + 服务端统一校验落地，不在 v1 内引入完整 IAM 平台

**Issues Resolved:**

- 统一了首页、趋势、分布、病种和下钻之间的时间窗口与筛选语义
- 打通了医生端实时评估结果到管理端下钻的持久化复用链路
- 为管理端问题病例复核提供了真实 CSV 导出和角色边界验证

**Issues Deferred:**

- 整体诊断质量指数的最终公式与校准来源仍需后续 milestone 明确
- 真实 EMR 集成的更深层宿主约束与生产级身份体系仍待后续演进
- 治理预警、跨院区对比、规则版本对比等 v2 能力未纳入 v1

**Technical Debt Incurred:**

- 角色上下文仍依赖宿主注入 header/context，尚未演进为完整认证授权体系
- milestone 归档前 live `REQUIREMENTS.md` 的 checkbox/traceability 未及时回写，本归档已按已验证交付结果做规范化修正
