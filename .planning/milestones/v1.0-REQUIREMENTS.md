## v1 Requirements

### Data Integration

- [x] **DATA-01**: 系统可以接入门诊病历、诊断结果和评估结果数据，并形成统一分析数据模型
- [x] **DATA-02**: 系统可以按日、周、月等统计口径聚合门诊量、评估病例数、诊断依据不完整比例和缺失诊断比例
- [x] **DATA-03**: 系统可以区分已评估与未评估病例，并支撑后续图表和下钻查询

### Dashboard Overview

- [x] **DASH-01**: 用户可以在首页查看默认时间范围及其起止日期，默认口径符合 PRD 约定
- [x] **DASH-02**: 用户可以切换近三个月、近半年、近一年、过去 30 天、过去 7 天、上周、上个月、上个季度、上一年和自定义时间
- [x] **DASH-03**: 首页概览卡可以展示本期值、与上期差值、增减符号和 N/A 规则
- [x] **DASH-04**: 首页概览卡至少展示门诊量、评估病例数、诊断依据不完整比例、缺失诊断比例和整体诊断质量指数

### Trend Analysis

- [x] **TREN-01**: 用户可以查看门诊诊断质量趋势图，按选定时间维度切换横坐标粒度
- [x] **TREN-02**: 趋势图可以同时展示门诊量、评估病例数、诊断依据不完整比例和缺失诊断比例
- [x] **TREN-03**: 用户悬停趋势图任意点位时可以查看时间和对应指标值

### Distribution Analysis

- [x] **DIST-01**: 用户可以查看按科室分布的气泡图，X 轴为门诊量，Y 轴为诊断依据不完整比例
- [x] **DIST-02**: 气泡图可以用气泡大小表示评估病例数，用颜色表示诊断缺漏比例
- [x] **DIST-03**: 用户可以按科室类型筛选分布图，并在悬停时查看科室详细数据

### Disease Insights

- [x] **DISE-01**: 用户可以查看高发问题病种词云，默认展示问题数量靠前的病种
- [x] **DISE-02**: 词云颜色和布局可以体现病种问题严重程度，并遵循 PRD 中的排序逻辑

### Realtime Evaluation

- [x] **EVAL-01**: 门诊医生可以在病历界面一键触发诊鉴评估
- [x] **EVAL-02**: 系统可以在约 10 秒内返回诊断依据完整性和缺漏诊断评估结果
- [x] **EVAL-03**: 评估结果需要包含具体评估依据和可操作建议，便于医生补充病历或调整诊断
- [x] **EVAL-04**: 系统需要记录单次诊鉴结果，供统计首页和病例下钻复用

### Drilldown and Export

- [x] **OPER-01**: 管理者可以从首页下钻到科室、医生或病种维度的问题明细
- [x] **OPER-02**: 管理者可以查看并下载问题病例，支持进一步人工复核
- [x] **OPER-03**: 系统需要对不同角色开放相应查询能力，避免医生和管理端权限混淆
