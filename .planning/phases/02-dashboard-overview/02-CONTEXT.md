<decisions>
## Implementation Decisions

### 时间筛选体验
- **D-01:** 首页顶部采用“快捷时间范围选择 + 自定义时间入口”的组合方案，而不是自由输入优先。
- **D-02:** 快捷时间范围直接覆盖 PRD 中的常用范围：近三个月、近半年、近一年、过去30天、过去7天、上周、上个月、上个季度、上一年、自定义时间。
- **D-03:** 顶部需要始终展示当前起止日期与默认截止规则，文案明确到具体日期，而不是只显示相对时间名称。
- **D-04:** 默认加载仍严格继承 Phase 1：以“昨日”为截止口径，默认选中“近三个月”，展示 13 个按周聚合的时间单元。

### 概览卡片内容
- **D-05:** 首页概览卡固定展示 5 张卡，顺序为：门诊量、评估病例数、诊断依据不完整比例、缺失诊断比例、整体诊断质量指数。
- **D-06:** 每张卡采用“主值 + 较上期变化”结构，不增加额外说明文案作为主视觉层级，保持首页扫描效率。
- **D-07:** “门诊量”默认显示按就诊记录统计的口径，“评估病例数”默认显示成功评估病例数；不在首页卡片直接暴露备用统计口径。
- **D-08:** 整体诊断质量指数在业务公式未确认前，卡片位置保留，但默认展示占位状态（如 `待定义` / `--`），不展示伪造数值，也不展示环比。

### 数值与环比规则
- **D-09:** 数值展示规则严格按 PRD：比例默认百分比，保留两位小数并去除无意义尾零；整体诊断质量指数保留两位小数；数量指标按整数显示。
- **D-10:** 环比视觉规则严格沿用 PRD：诊断依据不完整比例、缺失诊断比例上升显示红色 `↑`、下降显示绿色 `↓`；门诊量、评估病例数、整体诊断质量指数增加显示红色 `+`、减少显示绿色 `-`。
- **D-11:** 上期值为 0、上期值缺失、或当前不具备可比性的情况统一显示 `N/A`，而不是硬算百分比变化。
- **D-12:** 较上期完全无变化时统一显示 `较上期 --`，不显示颜色强调。

### 趋势图表现与交互
- **D-13:** 趋势图采用 PRD 指定的双轴组合图：右轴为门诊量和评估病例数，左轴为诊断依据不完整比例与缺失诊断比例。
- **D-14:** 门诊量与评估病例数采用“灰色总门诊量 + 蓝色评估病例数”的重叠/遮罩柱表现，突出已评估与未评估关系，而不是并列双柱。
- **D-15:** 趋势图默认同时展示两条比例折线，不提供单线默认简化态；分析首页优先完整表达质量趋势。
- **D-16:** Tooltip 必须同时展示当前时间点的时间标签、门诊量、评估病例数、诊断依据不完整比例、缺失诊断比例，并保持白底浅灰边框风格。
- **D-17:** 时间粒度切换完全继承 Phase 1 的 SQL 语义层，不允许前端自行推导时间桶或改写统计口径。

### 空态与异常态
- **D-18:** 当所选时间范围无数据时，概览卡与趋势图都展示明确空态，不用 `0` 冒充真实统计结果。
- **D-19:** 当存在数据但某个指标无法计算时，页面按单指标缺失处理，保留其他可展示指标，不让整个模块失败。

### the agent's Discretion
- 快捷时间范围控件是分段按钮、胶囊标签还是下拉组合，只要不改变上面已锁定的交互原则即可。
- 卡片图标、间距、字体层级和趋势图的具体视觉样式。
- Tooltip 排版细节、默认 hover 命中策略和图例位置。

</decisions>

- `sql/migrations/002_phase1_metrics.sql`：已经提供默认时间窗口、概览汇总和趋势序列 SQL 语义层，可直接支撑 Phase 2 页面读取。

### Established Patterns
- 当前代码以“SQL 语义层 + Python 查询层”作为既定模式，前端或页面层不应重新计算时间窗口、环比或比例。
- 指标与时间桶口径已在 Phase 1 中锁定，Phase 2 应专注展示规则与交互，不重新定义统计语义。

### Integration Points
- Phase 2 将直接消费 Phase 1 的 `analytics.get_overview_summary` 与 `analytics.get_trend_series` 结果。
- 未来 Phase 3 的分布图和词云仍会复用相同筛选与时间口径，因此 Phase 2 的时间筛选体验需要具备可延展性。

<specifics>
## Specific Ideas

- 时间选择器应优先让管理者快速切换常用范围，而不是先暴露复杂自定义交互。
- 首页概览卡要保留“整体诊断质量指数”卡位，但在业务公式未锁定前只展示占位状态，不伪造指标值。
- 趋势图优先完整表达 4 个核心指标，不做默认简化成单线或单指标模式。
- 空态要明确表达“当前范围无数据”，不能把“无数据”和“零值”混淆。

</specifics>
